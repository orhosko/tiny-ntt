\section{Introduction and Project Summary}
This section introduces the term project by summarizing the main characteristics of the system to be implemented. Student may include very rough technical details but rather the main problem and how the designed system is solves the problem is to be defined in this section. This section is expected not to exceed two or three pages.
\newpage

\section{Comparative Literatre Survey}
In this section you should state all the \textbf{related work that you have used in your final design}. For each study, you should compare how your system compares qualitatively with the literature work. You should heavily use citations during your survey. An example is provided below: \\

In November 1936 Alan Turing published his groundbreaking paper which creates foundations of computer science \cite{turing1937computability}. It was the Big-Bang of the computer science that created a new universe from dusty works of his ancestors.\\

A good website to search for academic references is google scholar. You should be able to access the papers that you've searched if you use an ITU IP address.
\newpage

\section{Developed Approach and System Model}
In this section you should present the model of your prepared system. You should present models of your system using various aspects such as data model, behavioral model, algorithms’ structure, interface. You might include additional model sor exclude non-relevant ones according to your design. For instance a mobile application project might contain a lot of screen mock-ups where a network protocol desing should contain a lot of state machine diagrams and pseudocode.
\subsection{Data Model}
You should present static data structures in this section. Database E-R diagrams, file format or class diagrams are good candidates.
\subsection{Structural Model}
Present the key algorithms, hardware design and/or developed approach in this section. It is expected that you will be using algorithm definitions and pseudocode.
\subsection{Dynamic Model}
This section should contain dynamic models (both overall and component based) based on your system behaviour. UML seuquence diagrams, statecharts and state machines are some examples that you can use. You might also present the flow between the interface mockups.

\newpage
\section{Experimentation Environment and Experiment Design}
If your project contains experiments during the evaluation of your approach you should include the details of your experimentation environment. This section can introduce your development environment and resource requirements of your final design if there doesn’t exist any substantial experimentation in your project. Another possibility is to introduce your unit/functional/performance tests in this section for software intensive projects.

\newpage
\section{Comparative Evaluation and Discussion}
You should evaluate your system design in this section. How does your system compare with the approaches in the literature in terms of quantitative measurement. Evaluation criteria might change according to your system characteristics but you should present some kind of evaluation based on quantitative results. You should reference the goals and evaluation criteria presented in your interim report, discuss if your system is able meet them.

\newpage
\section{Conclusion and Future Work}
You should summarize your project and present final remarks. It is also crucial that you include future work and possibilities to extend your design in the future.
